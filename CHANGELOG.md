### Fixed
- **Ctrl+M (CycleModel) and Shift+Enter (InsertNewline) don't work without Kitty keyboard protocol** — Terminals without Kitty protocol support (GNOME Terminal, xterm, SSH sessions) send `Ctrl+M` and `Shift+Enter` as byte `0x0D`, identical to plain `Enter`. Both keybindings now have `Alt+` fallback alternatives: `Alt+M` for CycleModel, `Alt+Enter` for InsertNewline. Alt always sends a distinct `ESC` prefix, so it works on every terminal. On macOS, `⌥m` (which produces `µ`) is also handled. Hint text in the prompt border and session chrome adapts automatically — shows `Alt+Enter`/`Alt+M` when Kitty protocol is unavailable, `Shift+Enter`/`Ctrl+M` when it is. Modified: `src/tui/keybindings/bindings.rs`, `src/tui/keybindings/hints.rs`, `src/tui/draw_input.rs`, `src/tui/draw_output/session_chrome.rs`, `src/app/state/app.rs`, `src/tui/run.rs`.
- **`test_notif` example fails to compile on Linux** — `examples/test_notif.rs` used `std::os::windows::process::CommandExt` and `creation_flags()` unconditionally, causing build failure on non-Windows. Added `#[cfg(windows)]` / `#[cfg(not(windows))]` gated `main()` functions so it compiles as a no-op on Linux/macOS. Modified: `examples/test_notif.rs`.
- **Context badge stuck at 100% with few user messages** — Sessions that crossed the 400K compaction threshold with ≤3 user messages since last compaction could never compact because `compaction_boundary(keep=3)` requires >3 user messages to find a split point. `spawn_compaction_agent()` now tries progressively smaller `keep` values (3 → 2 → 1), ensuring compaction can run as long as at least one user message boundary exists. Modified: `src/tui/event_loop/agent_events.rs`.

- **Session JSONL files not deleted after compaction or prompt supersede** — When `store_append_from_display()` consumed events from memory (during compaction or when a new prompt killed the running process), it removed the `pid_session_target` entry but left the JSONL file on disk. Added the flag to the MSVC branch of CUDA host compiler options in the ggml-cuda CMakeLists. Modified: `vendor/whisper-rs/sys/whisper.cpp/ggml/src/ggml-cuda/CMakeLists.txt`.

### Fixed
- **GitView actions pane clipping stash and auto actions** — The actions pane height was hardcoded to 8 (main) / 10 (feature branch) rows, which didn't account for the stash/stash-pop actions added recently. Now computed dynamically from `git_actions_labels().len()` + extra rows (divider, auto-resolve, auto-rebase on feature) + 2 border, so the pane always sizes to fit. Modified: `draw_sidebar.rs`.
- **Auto-rebase not triggered after pull or squash merge** — After pulling on main or completing a squash merge (direct or via RCR), worktrees with auto-rebase enabled had to wait up to 2 seconds for the next periodic check. Now the auto-rebase timer is reset immediately on: successful pull (detected via "Pulled:" prefix in `GitResult`), squash merge success, and RCR completion with merge. Modified: `git_polling.rs`.
- **Auto-rebase only processing one worktree per tick** — `check_auto_rebase()` returned after the first successful rebase, so only one worktree was rebased every 2 seconds. Combined with a single-slot notification (`Option<(String, Instant)>`), the user only saw the last toast. Rewrote to process ALL eligible worktrees in a single pass, collecting results into a `Vec<String>`. A combined 3-second toast lists every rebased branch. Conflicts still halt processing (needs user intervention). Modified: `auto_rebase.rs`, `app.rs` (field type change), `overlays.rs` (multi-line dialog), `run.rs` (call site).
- **Shift+Tab not cycling panes backward** — `KeyCombo::matches()` only accepted `BackTab` with `KeyModifiers::NONE`, but some terminals deliver Shift+Tab as `(SHIFT, BackTab)`. The shifted-symbol fallback only handled `Char` keys, so BackTab with SHIFT modifier fell through. Added explicit `BackTab` case to the shifted-symbol fallback. Modified: `types.rs`. 2 tests added.
- **Deferred render expansion loop preventing scroll to top** — When a user scrolled to the top of a large conversation (200+ events), the deferred render expansion reset `rendered_events_start` and `rendered_events_count` to 0. On the next frame, `submit_render_request()` saw those same zeros and re-triggered deferral (skipping early events again), creating an infinite loop. The user could never see early messages and the message counter showed incorrect positions. Fixed by adding an `expanding_deferred` guard that only allows deferral on initial fresh loads, not on expansion renders. Modified: `render_submit.rs`. 1 regression test added.
- **Live session overriding explicit session list selection** — When a Claude process was running and the user selected a *different* session from the session list, `load_session_output()` always took the live JSONL path (because `is_live` was true), ignoring the user's selection. The historic session never loaded — the pane showed the live session instead. Fixed by detecting when `session_selected_file_idx` points to a store session ID different from the active slot's `pid_session_target` ID, and forcing the historic/store load path for that case. Modified: `load.rs`.
- **Context badge under-reporting on worktree switch-back** — When switching back to a worktree with an active session, `update_token_badge()` read only store chars, but the current turn's JSONL events (not yet ingested into SQLite) were not counted. The badge showed a lower percentage than actual context usage, or showed a stale value instead of the correct 100%. Fixed by computing `event_char_len()` for all parsed JSONL events before they are consumed, then adding that total to `chars_since_compaction` after the store sync. Modified: `load.rs`.

### Fixed
- **Global keybindings not working in terminal command mode** — Single-letter globals (`G`, `H`, `M`, `P`, `]`/`[`, `r`, etc.) were blocked during terminal command mode because the skip guard in `lookup_action()` checked `terminal_mode && focus == Input` which was always true when the terminal pane was focused. Narrowed the skip to only block during terminal *type mode* (`prompt_mode=true`), allowing all globals in command mode. Modified: `lookup.rs`.
- **`p` not refocusing prompt when tabbed away** — Pressing `p` when the prompt input was open but focus was on another pane (e.g. file tree, session) did nothing because `EnterPromptMode` was grouped with all single-letter globals in the skip guard that blocks when `prompt_mode=true`. Gave `EnterPromptMode` its own narrower skip — only blocked in edit mode or when prompt is already focused (`focus == Input`). Modified: `lookup.rs`.
- **Prompt text fragments stuck in command box after submit** — After submitting a prompt, fragments of the typed text sometimes remained visible in the input area (now in command mode). Root cause: `fast_draw_input()` writes directly to the terminal via crossterm, bypassing ratatui's internal buffer. When `clear_input()` + `prompt_mode=false` fired on Enter, fast_draw stopped running (eligibility check fails), but ratatui's diff engine saw no changes between its stale "previous" buffer and the new empty-input buffer — so it never overwrote the physical terminal cells. Fixed by snapshotting fast-draw eligibility before key processing and running one final reconciliation `fast_draw_input()` when eligibility transitions from true→false, clearing stale content with padded spaces. Modified: `event_loop.rs`.

(2) `close_git_actions_panel()` only called `rebase_abort()` on the feature branch, never cleaning up squash merge state on main (repo root). Added `cleanup_squash_merge_state()` call that resets unmerged files and pops the orphaned stash. (3) The conflict overlay's abort handler (`n`/`Esc`) had the same feature-branch-only cleanup. - **Session content bleeds across sessions on switch** — Three isolation failures when switching between sessions: (1) `rendered_lines_cache` was only marked dirty, not cleared, so stale rendered lines from the previous session flashed for 1-2 frames until the render thread completed new work; in-flight render results from the old session could also be applied after the switch. (3) The PID badge in the convo border showed the active slot's PID regardless of which session file was being viewed, making finished sessions appear active. Fixed by suppressing PID/exit display when `viewing_historic_session` is true.

Press `s` in the Git panel to open an interactive overlay for managing the auto-resolve file list (j/k navigate, Space toggle, `a` add, `d` remove, Esc save). Config persisted per-project in azufig.toml. Replaces the hardcoded `DOC_FILES` constant and `--theirs` strategy.
- **Rebase stashes dirty working tree** — `exec_rebase_inner()` now runs `git stash --include-untracked` before rebasing and pops on all exit paths (success, failure, conflict). Prevents "cannot rebase: You have unstaged changes" from files like `.DS_Store` or editor swap files that aren't in the git panel's changed files list.
- **Squash merge stash checks exit code** — `squash_merge_into_main()` stash now validates `status.success()` in addition to checking stdout text, preventing silent stash failures from leaving dirty state that blocks `git merge --squash`.
- **Worktrees show as archived during RCR and after detached HEAD** — During RCR, the rebase leaves HEAD detached. If `load_worktrees()` ran, it saw `branch: None` → blank sidebar entry + real branch appended as archived ghost, and output routing broke (convo pane showed nothing). Fixed with two-case handling: (1) active rebase → reads branch from `rebase-merge/head-name` without aborting; (2) orphaned detached HEAD → re-attaches via checkout. Also moved repair from startup-only into `load_worktrees()` so every refresh benefits. Belt-and-suspenders: `handle_claude_output()` now always shows output matching the RCR session's slot ID, regardless of `current_worktree().branch_name`.

### Changed
- **Centralized branch prefix** — All 9 hardcoded `"azureal/"` branch prefix assumptions replaced with a `BRANCH_PREFIX` constant and `strip_branch_prefix()` helper in `src/models.rs`, making the app fully repo-agnostic. Doc comments updated to use generic `{prefix}/` examples.
- **azufig confirmed project-scoped** — Removed aspirational worktree-local azufig documentation (never implemented). The project-local `.azureal/azufig.toml` is always at the main worktree root (resolved via `git rev-parse --git-common-dir` parent), shared by all worktrees. No per-worktree copies.
- **Git status box hints** — Removed `a:auto-rebase` from status box footer since it's already listed in the Actions pane.
- **`.DS_Store` untracked from git** — File was in `.gitignore` but already tracked in git history, causing it to appear as a conflict offender during RCR. Removed from tracking via `git rm --cached`.
- **Auto-untrack gitignored files** — `Git::untrack_gitignored_files()` runs `git ls-files -i --exclude-standard` to find tracked files matching `.gitignore`, then `git rm --cached` to remove them from the index without deleting the working-tree copy. Called on startup (`load()`) and after every `stage_all()` (`git add -A`) to prevent gitignored files from being committed or causing rebase/merge conflicts.

### Removed
- **AZUREAL++ panel** — Entire panel removed (⌃a keybinding, GitHub issues/PRs integration, debug tab). Debug dump kept as standalone `⌃d` shortcut.
- **Auto-rebase of peer worktrees** — Removed automatic rebase of peer worktrees after squash merge. Manual rebase (`r`) replaces it — auto-rebase wastes resources when not needed.
- **`pre_merge_head` from RcrSession** — No longer needed since RCR uses `git rebase --abort` (which handles state restoration) instead of `git reset --hard`.
- **1,562 lines of dead code** — Removed unused types (`ContextMenu`, `WorktreeAction`, `ViewMode::Diff/Messages/Rebase`, `RebaseState`, `RebaseStatus`), dead methods (`load_diff`, `scroll_diff_*`, `destroy_terminal`, `session_has_terminal`), the entire `InteractiveSession` system (`spawn_interactive`, `send_prompt`), unused git ops (`fetch`, `merge_abort`, `rebase_onto_main`, 6 more rebase helpers), `DiffHighlighter` (syntect-based, replaced by Viewer pane), `DisplayEvent::Error`, `ClaudeEvent::Error`, and all orphaned match arms/imports. Compiler now produces zero warnings.
