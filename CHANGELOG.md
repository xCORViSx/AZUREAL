### Added
- **Health panel auto-refresh** — God Files and Documentation tabs automatically rescan when source files change in the worktree (500ms debounce). Preserves tab, cursor positions, scroll offsets, and checked states across rescans. No manual re-open needed after editing files.
- **Unified Add Worktree dialog (`a`)** — Merges the old standalone branch browser (`b`) into the Add Worktree action. Dialog shows a "[+] Create new" row at the top (uses filter text as worktree name) with branches listed below. Branches show `[N WT]` worktree count indicator instead of `[active]`. Filter input doubles as new worktree name with git-safe character validation. Enter on a checked-out branch switches to it; Enter on an unchecked branch creates a worktree from it.
- **Delete worktree sibling guard (`⌘d`)** — When deleting a worktree whose branch has other worktrees (siblings), shows a dialog: "N other worktree(s) on this branch. Delete all? (y) Archive only? (a)". `y` removes all sibling worktrees then deletes the branch. `a` archives the current worktree only (removes directory, keeps branch). Sole worktrees get the standard y/N delete+branch confirmation. This respects git's constraint that branches can't be deleted while worktrees are checked out.
- **Remote branch deletion** — `Git::delete_branch()` now also runs `git push origin --delete` (best-effort) to clean up remote branches when deleting worktrees.

### Changed
- **God file detection excludes test modules** — For Rust files, `#[cfg(test)]` module blocks are now excluded from line counts via brace-depth tracking. This prevents files with large test suites from being falsely flagged as god files when the production code is well under the 1000-line threshold.
- **Worktree tab cycling wraps around** — `[`/`]` now wraps from the last worktree back to the first (and vice versa) instead of stopping at the boundary.
- **Worktree tab row focus indicator** — When focused (via Tab cycling), separators turn AZURE and inactive tab text brightens from DarkGray to Gray, matching the AZURE focus cue used by all other panes.
- **Worktree name removed from file tree pane title** — Title now shows `Filetree` instead of `Filetree (worktree-name)`. The worktree tab bar already indicates the active worktree, so the redundant label was removed.

### Removed
- **Standalone branch browser (`b`)** — Merged into the unified Add Worktree dialog (`a`). The `BrowseBranches` action and `b` keybinding are removed.
- **Worktree creation modal** — The standalone name-input modal (`Focus::WorktreeCreation`) is removed. Creating new worktrees now goes through the unified Add Worktree dialog's "[+] Create new" row.

- **Worktree sidebar replaced by horizontal tab row** — The vertical worktree sidebar (15% left column) is now a 1-row horizontal tab bar at the top of the normal mode layout. `[★ main]` tab always first (yellow when active, Shift+M toggles main browse). Active worktree tab uses AZURE bg + white fg + bold. Archived worktrees shown dim with `◇` prefix. Supports greedy pagination with `N/M` page indicator. Mouse click on tabs selects worktrees. `[`/`]` switch tabs globally from any pane; `←/→` switch when focused.
- **FileTree is always visible** — No longer a toggle overlay (`f` key removed). FileTree is a permanent pane in the 15% left column. Included in the focus cycle as a separate pane.
- **Focus cycle keys restored to Tab/Shift+Tab** — Pane cycling reverted from `{`/`}` (Shift+[/Shift+]) back to `Tab`/`Shift+Tab`. Cycle: Worktrees → FileTree → Viewer → Session → Input → Worktrees.
- **Keybindings relocated to GLOBAL** — `Shift+M` (BrowseMain), `Shift+P` (OpenProjects), `⌘r` (RunCommand) now work from any pane. `[`/`]` are global worktree switching keys.
- **Removed keybindings** — `⌃r` (Restart), `f` (ToggleFileTree), `Enter` (StartResume in worktrees), `/` (SearchFilter in worktrees). Sidebar filter removed entirely.

### Changed
- **Auto-push after rebase and squash merge** — All rebase and squash merge paths now auto-push to remote. Standalone rebase (`r`) pushes the rebased branch. Squash merge (`m`) pushes both the rebased feature branch and main. Auto-rebase pushes the branch after each successful rebase. RCR acceptance pushes in both cases. Status shows `"→ pushed"` on success or `"(push failed: ...)"` / `"(branch push failed: ...)"` / `"(main push failed: ...)"` on error. No longer requires manual push steps.
- **Worktree tab leading space removed** — Tab labels no longer have a leading space before the icon/symbol. Format changed from `" ● name "` to `"● name "` (trailing space only for separator padding). Tightens tab rendering and saves 1 char per tab.
- **Post-merge dialog centered on full screen with git colors** — Dialog now renders center-screen (was anchored to session pane) and uses `GIT_ORANGE` border/title (was AZURE) to match the git panel theme.
- **Git status box title simplified** — Title changed from `" GIT: <branch> "` to just `" GIT "` since the worktree tab bar already shows which branch is active.
- **Push label updated** — Git panel push action description changed from "Push" to "Push to remote" for clarity.
- **Model switcher always shows specific model** — `selected_model` is now always set (never None), initialized to "opus". Cycles through opus → sonnet → haiku → opus. The displayed name is exactly what gets passed as `--model` to `spawn()`. No session file detection or stream lookup needed.
- **Keybindings module decomposed** — `keybindings.rs` (1007 lines) split into 5 focused submodules: `types.rs` (KeyCombo, Action, Keybinding, HelpSection), `bindings.rs` (~21 static arrays + alt key statics), `lookup.rs` (KeyContext + lookup_action + 7 per-modal lookups), `hints.rs` (help_sections, title/hint generators, find_key_for_action), `platform.rs` (macos_opt_key). Module root re-exports all public items — no import changes needed in consuming files.

### Fixed
- **Worktree tab row not updating after delete/archive/external changes** — Three issues: (1) `load_worktrees()` reset the selected tab to cwd-based on every refresh, causing the selection to jump after archive/delete. Fixed by preserving the current selection by branch name across refreshes. (2) `archive_current_worktree()` didn't reload session output after refresh, leaving stale content in the session pane. Fixed by calling `load_session_output()`. (3) Filesystem changes from external tools (embedded terminal, other processes) only refreshed the file tree, not the worktree tab list. Fixed by adding `worktree_tabs_refresh_pending` flag on `WorktreeChanged` events with debounced `refresh_worktrees()` call (500ms).
- **File tree retains scope UI after exiting health scope mode** — After pressing Esc to exit scope mode, the file tree continued showing green highlights and scope border until the user moved the cursor. The cached file tree render wasn't being invalidated when `god_file_filter_mode` was cleared. Fixed by calling `invalidate_file_tree()` in the escape handler.
- **External worktrees pollute tab row** — `git worktree list` returns ALL worktrees including ones created by external tools (e.g. Claude Code subagents creating worktrees in `.git/worktrees/` or `.claude/worktrees/`). These appeared in the tab row as unexpected entries. Fixed by filtering `load_worktrees()` to only include worktrees whose path is under the project's `worktrees_dir()` (`<repo_root>/worktrees/`), plus the main worktree. Detached HEAD repair also scoped to owned worktrees only.
- **Divergence indicator shows stale ↑N after squash merge** — After squash merge, `git rev-list main...HEAD` on the feature branch still showed N commits ahead because squash merge creates a different commit (not a merge commit linking histories). Fixed by fast-forwarding the feature branch to main (`git reset --hard main`) after successful squash merge, so divergence shows 0/0 if the branch is kept.
- **Unarchiving a worktree creates a ghost archived duplicate** — `create_worktree_from_branch` used `branch_name.contains('/')` to detect remote branches, but `azureal/foo` is a local branch with a `/`. This caused `git worktree add --track -b foo <path> azureal/foo`, creating a new branch `foo` while `azureal/foo` persisted as an orphaned archived entry. Fixed by checking `refs/heads/<branch>` to detect local branches before falling back to the remote-tracking path.
- **Health scope shows no green highlights in non-main worktrees** — Scope directories were persisted as absolute paths rooted at `project.path` (main repo root, e.g., `/repo/src`), but the file tree displayed entries from the current worktree path (`/repo/worktrees/health/src`). The `HashSet::contains()` check always failed for non-main worktrees because the paths never matched. Fixed by translating project-root paths → worktree paths when entering scope mode, and worktree paths → project-root paths when saving on exit.
- **Session content bleeds across sessions on switch** — Three isolation failures when switching between sessions: (1) `rendered_lines_cache` was only marked dirty, not cleared, so stale rendered lines from the previous session flashed for 1-2 frames until the render thread completed new work; in-flight render results from the old session could also be applied after the switch. Fixed by clearing all render caches immediately and advancing `render_seq_applied` to discard stale render thread results. (2) When viewing a different session file via the session list while a Claude process was running on the same branch, live events from the running process were appended to the viewed session's `display_events` because `handle_claude_output` only gated on branch + active slot, not on which session file was selected. Fixed by adding `viewing_historic_session` flag that suppresses live event routing when the viewed session UUID doesn't match the active slot's session. (3) The PID badge in the convo border showed the active slot's PID regardless of which session file was being viewed, making finished sessions appear active. Fixed by suppressing PID/exit display when `viewing_historic_session` is true.

### Added
- **Unread session indicator (`◐`)** — When a Claude session finishes on a worktree the user isn't currently viewing (different branch, or same branch but a background slot), the tab shows a half-filled circle `◐` in AZURE instead of the normal status symbol. Works per-session: if any one session in the session list is unread, the tab shows `◐` — even if other sessions are still running (unread takes priority over running `●`). Unread clears per-session when that specific session is viewed, and the branch-level `◐` only disappears when all sessions on the branch have been viewed. Clears only when the session pane is visible (normal mode or git panel close) — switching worktrees inside the git panel preserves unread state. Two-level tracking: `unread_sessions: HashSet<String>` (branch names, for tab rendering) and `unread_session_ids: HashSet<String>` (session UUIDs, for per-session granularity). Populated in `handle_claude_exited()`, cleared in `load_session_output()` and `close_git_actions_panel()`.
- **Auto-rebase `R` indicator in worktree tabs** — Both the normal worktree tab row and the git panel tab row now show a bold `R` suffix next to worktrees with auto-rebase enabled. Color-coded: green=idle, orange=RCR active (conflict resolution running), blue=RCR approval pending. Computed via `rebase_indicator_color()` helper that checks `auto_rebase_enabled`, `rcr_session.branch`, and `rcr_session.approval_pending`.
- **Quit blocked during git operations** — `⌃q` is blocked when any git operation is in progress (commit, push, RCR conflict resolution, commit message generation). Shows "Cannot quit while a git operation is in progress" in the status bar. Prevents repo corruption from interrupted git commands.
- **Delete worktree shortcut (`⌘d`)** — Press `⌘d` from the worktree tab row to permanently delete the current worktree and its git branch (both local and remote). Shows a y/N confirmation in the status bar. Cleans up auto-rebase config on deletion.
- **Branch divergence indicators** — Git panel commits pane bottom border shows compact badges for main and remote divergence. Main badge: `↑N ↓M main` (red bg when behind, green when only ahead; feature branches only). Remote badge: `↑N ↓M remote` (yellow bg when behind, cyan when only ahead). Both computed via `git rev-list --left-right --count` on panel open and after refresh operations. Replaces the old behind-main-only indicator.
- **Git panel worktree tab bar** — A 1-row horizontal tab bar at the very top of the Git panel, using the same design as the normal worktree tab row: `★ main` tab always first, status symbols (●/◇), archived styling, greedy pagination, and mouse click support. Colors use the git palette: active worktree `GIT_ORANGE` bg + white fg + bold; active ★ main `GIT_ORANGE` bg + black fg + bold; inactive `GIT_BROWN` fg; separators `GIT_BROWN │`. Clicking ★ main (or Shift+M) opens the main branch's git view. `[`/`]` cycle worktrees (skips main, matching normal mode), `{`/`}` jump pages. Focused pane preserved across switches.
- **Model switcher (`⌃m`)** — Press `⌃m` from any pane to cycle the Claude model: sonnet → opus → haiku. Session pane bottom border shows a right-aligned `⌃m:model` indicator color-coded per model (magenta=opus, cyan=sonnet, yellow=haiku). `selected_model` is always set and always passed as `--model` to `spawn()`. Works from both command mode and prompt input mode. Automated spawns (RCR, health panel) always use the CLI default (pass `None`).
- **Git panel text selection and copy** — The git diff viewer now supports mouse drag selection, `⌘A` (select all), and `⌘C` (copy) — same as the normal viewer. `draw_git_viewer_selectable` populates `viewer_lines_cache` with styled diff lines so the existing selection infrastructure works seamlessly. `⌘C` falls back to copying the status box result message when no viewer selection exists. Git diffs use gutter=0 (no line numbers to strip). `⌘C`/`⌘A`/`Shift+J`/`Shift+K`/`PageDown`/`PageUp` intercepted early in `handle_git_actions_input` before `lookup_git_actions_action()` since the git panel consumes all input.
- **Git status box text selection** — Click the git status box (bottom bar showing operation results) to select its text, shown with a selection highlight. `⌘A` selects the status box message when no viewer content exists. `⌘C` copies the selected text to clipboard. Selection clears on panel close, other pane clicks, or non-navigation keystrokes.
- **Git panel diff viewer scrolling** — `Shift+J`/`PageDown` and `Shift+K`/`PageUp` scroll the diff viewer by page in the Git panel.
- **Git actions pane divider** — Actions pane now shows a `─` horizontal rule between action items and the toggle section (auto-rebase, auto-resolve), visually separating operations from settings. Both main and feature branch layouts include the divider.
- **Compaction inactivity watcher** — When context usage reaches 95%+ and 20 seconds pass with no new session events, a yellow "Session may be compacting context..." banner appears in the session pane. Since Claude's auto-compaction produces no visible event until it completes, this heuristic prevents users from thinking the UI is frozen. Banner clears automatically when new events arrive.

### Fixed
- **Post-merge dialog resets worktree selection to top** — After choosing keep/archive/delete in the post-merge dialog, `refresh_worktrees()` reset `selected_worktree` based on cwd (always index 0). Now saves the current branch name before refresh and restores the selection to that branch, or clamps to the previous index if the worktree was removed.
- **Viewer placeholder text bleeds through content** — Ratatui buffer persistence caused old cells to show through when new content didn't fill the same area. The git viewer placeholder ("Select a file or commit...") bled through the commit editor, conflict overlay, auto-resolve overlay, and normal viewer content. Fixed by rendering `Clear` before `Paragraph+Block` in `draw_viewer.rs`, `draw_git_viewer_selectable`, `draw_commit_editor`, `draw_auto_resolve_overlay`, and `draw_conflict_inline`.
- **Worktree appears archived with detached HEAD on startup** — Orphaned rebase cleanup (`git rebase --abort`) ran AFTER `load_worktrees()`, and `--onto` rebases can leave HEAD detached after abort. Detached worktrees have no `branch` in `git worktree list --porcelain`, so they got empty branch names and appeared as archived duplicates. Fixed by moving cleanup BEFORE `load_worktrees()`, using `git symbolic-ref --quiet HEAD` to detect detached state, and re-attaching via `git for-each-ref --points-at=HEAD` → `git checkout <branch>` (repo-agnostic, no hardcoded branch prefix). This also prevents the working tree reset that was silently discarding uncommitted changes.
- **Mouse click in Git panel exits the panel** — `handle_mouse_click` had a blanket dismissal (`git_actions_panel = None`) leftover from when the panel was a centered modal overlay. Now clicks anchor viewer selection for drag-to-select; other pane clicks are consumed without closing.
- **Session start fails in worktrees with `+` or other special characters** — Azureal's path encoding only replaced `/` with `-`, while Claude CLI v2.1+ replaces ALL non-alphanumeric characters with `-` (e.g., `AZUREAL++` → `AZUREAL--`). This mismatch caused Azureal to look for session files in the wrong `~/.claude/projects/` subdirectory, so `--resume` pointed to a session Claude couldn't find. Fixed by matching Claude CLI's exact `OP()` algorithm (`replace(/[^a-zA-Z0-9]/g, "-")`) in `encode_project_path()`. Startup migration (`migrate_project_dirs()`) auto-renames old-encoding directories to the new encoding.
- **Session pane not reset on new worktree creation** — Creating a new worktree left `session_file_path` pointing at the previous worktree's session file. The file watcher would then re-populate `display_events` from the stale file on the next poll cycle. Fixed by clearing `session_file_path`, `session_file_modified`, `session_file_size`, and `session_file_dirty` at the top of `load_session_output()`.
- **Worktree creation dialog rejects valid characters** — Input filter was limited to alphanumeric, `-`, `_`, `.` only. Expanded to also allow `+`, `@`, `/`, `!` — all valid in git branch names.
- **Worktree creation dialog leaks keybindings** — Typing `G` (Shift+G) in the worktree name input opened the Git panel instead of inserting the character. Fixed by routing `BranchDialog` focus mode directly to its handler before keybinding resolution (standalone WorktreeCreation modal replaced by unified Add Worktree dialog).
- **Auto-rebase skips dirty worktrees** — `check_auto_rebase()` now checks `git status --porcelain` before rebasing; worktrees with uncommitted changes are silently skipped instead of failing.
- **Auto-rebase success dialog hidden while viewing another worktree's git panel** — `check_auto_rebase()` blanket-skipped all worktrees when the git panel was open. Now only skips the specific worktree whose git panel is being viewed — other worktrees still auto-rebase and show the 2-second success toast.
- **Orphaned rebase state cleaned up on startup** — On launch, all worktrees are scanned for `.git/rebase-merge/` or `.git/rebase-apply/` directories (left by app crashes during rebase) and auto-aborted.
- **Stash orphaned after merge-triggered RCR** — `accept_rcr()` now pops any stash left on main before re-calling `squash_merge_into_main()`, preventing orphaned stash entries.
- **Auto-rebase config not cleaned up on worktree removal** — Post-merge dialog archive/delete handlers now remove the `auto-rebase/<branch>` key from `azufig.toml` and the in-memory `HashSet`.
- **Push targets wrong branch / non-fast-forward rejections** — `Git::push()` now always resolves `HEAD` explicitly and runs `git pull --rebase origin <branch>` before pushing, preventing both wrong-upstream pushes and non-fast-forward rejections. Rebase conflicts are reported as errors; offline/no-upstream failures are silently skipped (push will create the branch). `Git::pull()` falls back to `git pull origin <branch>` when no upstream is configured. After rebase, `push()` detects diverged branches (`rev-list --left-right --count`) and auto-switches to `--force-with-lease`, skipping the pull-rebase step. Status message shows "(force-pushed)" suffix.
- **Squash merge fails when local main has unpushed merges** — `git pull --ff-only` on main during squash merge would bail with "Cannot update main" if local main diverged from remote (e.g., prior squash merge not yet pushed). Made pull always non-fatal — the feature branch is already rebased onto main, so the squash merge works regardless.
- **Squash merge fails with "unmerged files"** — Orphaned `UU` conflict markers on main (from prior interrupted merges or concurrent worktree work) blocked `git merge --squash`. Pre-flight cleanup now: aborts merge/rebase if state files exist, then resolves any remaining unmerged files with `checkout --ours` + `git add` so the squash merge can proceed.
- **Squash merge fails when main has dirty working tree** — `squash_merge_into_main()` now stashes dirty state before merging and pops after commit.
- **Squash merge reports false failure on already-merged branches** — `git commit` prints "nothing to commit" to stdout, not stderr. The error check only read stderr, so the `nothing to commit` early-return never triggered — instead it bailed with `"Squash merge staged but commit failed: "`. Fixed by checking both stdout and stderr. Also cleans up stale `SQUASH_MSG` files (left by failed merges) during pre-flight and after the nothing-to-commit path.
- **Squash merge with uncommitted changes loses work** — `exec_squash_merge()` now blocks if the feature branch has uncommitted changes.
- **RCR session disappears after Claude exits** — Fixed by skipping session-file re-parse when the exiting slot's file doesn't exist in the current worktree's session directory.
- **Branch dialog showed no branches** — All branches were checked out in worktrees and filtered out. Fixed by showing all branches with checked-out indicators instead of filtering.
- **Branch dialog froze UI** — `list_remote_branches` did `git fetch --all --prune` (blocking network call). Replaced with `list_remote_branches_cached` that reads local cache only.
- **Branch dialog created wrong worktree type** — Used `Git::create_worktree` (creates new branch with `-b`) instead of `Git::create_worktree_from_branch` for existing branches.
- **Commit log showed all worktrees' commits** — `git log` without a range filter followed HEAD's full ancestry, including squash merge commits from other branches that were merged into main. Feature branches now use `git log main..HEAD` to show only branch-specific commits.
- **Rebase replayed squash merge commits from other branches** — Plain `git rebase main` could replay other branches' squash merge commits that ended up in the branch's history from a prior rebase. Now uses `git rebase --onto main <fork-point>` to replay only branch-specific commits.
- **Configurable auto-resolve conflicts via union merge** — Files that frequently conflict during rebase (docs, config, etc.) can now be auto-resolved using `git merge-file --union` which keeps BOTH sides' changes with zero content loss. Default list: AGENTS.md, CHANGELOG.md, README.md, CLAUDE.md. Press `s` in the Git panel to open an interactive overlay for managing the auto-resolve file list (j/k navigate, Space toggle, `a` add, `d` remove, Esc save). Config persisted per-project in azufig.toml. Replaces the hardcoded `DOC_FILES` constant and `--theirs` strategy.
- **Rebase stashes dirty working tree** — `exec_rebase_inner()` now runs `git stash --include-untracked` before rebasing and pops on all exit paths (success, failure, conflict). Prevents "cannot rebase: You have unstaged changes" from files like `.DS_Store` or editor swap files that aren't in the git panel's changed files list.
- **Squash merge stash checks exit code** — `squash_merge_into_main()` stash now validates `status.success()` in addition to checking stdout text, preventing silent stash failures from leaving dirty state that blocks `git merge --squash`.
- **Worktrees show as archived during RCR and after detached HEAD** — During RCR, the rebase leaves HEAD detached. If `load_worktrees()` ran, it saw `branch: None` → blank sidebar entry + real branch appended as archived ghost, and output routing broke (convo pane showed nothing). Also moved repair from startup-only into `load_worktrees()` so every refresh benefits. Belt-and-suspenders: `handle_claude_output()` now always shows output matching the RCR session's slot ID, regardless of `current_worktree().branch_name`.
