# Changelog

All notable changes to Azureal will be documented in this file.

## [Unreleased]

### Changed
- **Worktree sidebar replaced by horizontal tab row** ‚Äî The vertical worktree sidebar (15% left column) is now a 1-row horizontal tab bar at the top of the normal mode layout. `[‚òÖ main]` tab always first (yellow when active, Shift+M toggles main browse). Active worktree tab uses AZURE bg + white fg + bold. Archived worktrees shown dim with `‚óá` prefix. Supports greedy pagination with `N/M` page indicator. Mouse click on tabs selects worktrees. `[`/`]` switch tabs globally from any pane; `‚Üê/‚Üí` switch when focused.
- **FileTree is always visible** ‚Äî No longer a toggle overlay (`f` key removed). FileTree is a permanent pane in the 15% left column. Included in the focus cycle as a separate pane.
- **Focus cycle keys changed** ‚Äî `Tab`/`Shift+Tab` replaced with `{`/`}` (Shift+[/Shift+]). Cycle: Worktrees ‚Üí FileTree ‚Üí Viewer ‚Üí Session ‚Üí Input ‚Üí Worktrees.
- **Keybindings relocated to GLOBAL** ‚Äî `Shift+M` (BrowseMain), `Shift+P` (OpenProjects), `‚åòr` (RunCommand) now work from any pane. `[`/`]` are global worktree switching keys.
- **Removed keybindings** ‚Äî `‚åÉr` (Restart), `f` (ToggleFileTree), `Enter` (StartResume in worktrees), `/` (SearchFilter in worktrees). Sidebar filter removed entirely.

### Changed
- **Post-merge dialog centered on full screen with git colors** ‚Äî Dialog now renders center-screen (was anchored to session pane) and uses `GIT_ORANGE` border/title (was AZURE) to match the git panel theme.
- **Git status box title simplified** ‚Äî Title changed from `" GIT: <branch> "` to just `" GIT "` since the worktree tab bar already shows which branch is active.
- **Push label updated** ‚Äî Git panel push action description changed from "Push" to "Push to remote" for clarity.

The displayed name is exactly what gets passed as `--model` to `spawn()`. No session file detection or stream lookup needed.

### Fixed
- **Health scope shows no green highlights in non-main worktrees** ‚Äî Scope directories were persisted as absolute paths rooted at `project.path` (main repo root, e.g., `/repo/src`), but the file tree displayed entries from the current worktree path (`/repo/worktrees/health/src`). The `HashSet::contains()` check always failed for non-main worktrees because the paths never matched. Fixed by translating project-root paths ‚Üí worktree paths when entering scope mode, and worktree paths ‚Üí project-root paths when saving on exit.
- **Session content bleeds across sessions on switch** ‚Äî Three isolation failures when switching between sessions: (1) `rendered_lines_cache` was only marked dirty, not cleared, so stale rendered lines from the previous session flashed for 1-2 frames until the render thread completed new work; in-flight render results from the old session could also be applied after the switch. Fixed by clearing all render caches immediately and advancing `render_seq_applied` to discard stale render thread results. (2) When viewing a different session file via the session list while a Claude process was running on the same branch, live events from the running process were appended to the viewed session's `display_events` because `handle_claude_output` only gated on branch + active slot, not on which session file was selected. Fixed by adding `viewing_historic_session` flag that suppresses live event routing when the viewed session UUID doesn't match the active slot's session. (3) The PID badge in the convo border showed the active slot's PID regardless of which session file was being viewed, making finished sessions appear active. Fixed by suppressing PID/exit display when `viewing_historic_session` is true.

Since Claude's auto-compaction produces no visible event until it completes, this heuristic prevents users from thinking the UI is frozen. Banner clears automatically when new events arrive.

When enabled, the worktree is automatically rebased onto main every 2 seconds when main is ahead ‚Äî but only when no Claude agent is running, no file is being edited, and no RCR is active. On conflict, opens the conflict overlay with full RCR flow. Sidebar shows colored `R` indicator: green (enabled), orange (RCR active), blue (approval pending). ### Fixed
- **Post-merge dialog resets worktree selection to top** ‚Äî After choosing keep/archive/delete in the post-merge dialog, `refresh_worktrees()` reset `selected_worktree` based on cwd (always index 0). Now saves the current branch name before refresh and restores the selection to that branch, or clamps to the previous index if the worktree was removed.
- **Viewer placeholder text bleeds through content** ‚Äî Ratatui buffer persistence caused old cells to show through when new content didn't fill the same area. The git viewer placeholder ("Select a file or commit...") bled through the commit editor, conflict overlay, auto-resolve overlay, and normal viewer content. Fixed by rendering `Clear` before `Paragraph+Block` in `draw_viewer.rs`, `draw_git_viewer_selectable`, `draw_commit_editor`, `draw_auto_resolve_overlay`, and `draw_conflict_inline`.
- **Worktree appears archived with detached HEAD on startup** ‚Äî Orphaned rebase cleanup (`git rebase --abort`) ran AFTER `load_worktrees()`, and `--onto` rebases can leave HEAD detached after abort. Detached worktrees have no `branch` in `git worktree list --porcelain`, so they got empty branch names and appeared as archived duplicates. Fixed by moving cleanup BEFORE `load_worktrees()`, using `git symbolic-ref --quiet HEAD` to detect detached state, and re-attaching via `git for-each-ref --points-at=HEAD` ‚Üí `git checkout <branch>` (repo-agnostic, no hardcoded branch prefix). This also prevents the working tree reset that was silently discarding uncommitted changes.
- **Mouse click in Git panel exits the panel** ‚Äî `handle_mouse_click` had a blanket dismissal (`git_actions_panel = None`) leftover from when the panel was a centered modal overlay. Now clicks anchor viewer selection for drag-to-select; other pane clicks are consumed without closing.
- **Session start fails in worktrees with `+` or other special characters** ‚Äî Azureal's path encoding only replaced `/` with `-`, while Claude CLI v2.1+ replaces ALL non-alphanumeric characters with `-` (e.g., `AZUREAL++` ‚Üí `AZUREAL--`). This mismatch caused Azureal to look for session files in the wrong `~/.claude/projects/` subdirectory, so `--resume` pointed to a session Claude couldn't find. Fixed by matching Claude CLI's exact `OP()` algorithm (`replace(/[^a-zA-Z0-9]/g, "-")`) in `encode_project_path()`. Startup migration (`migrate_project_dirs()`) auto-renames old-encoding directories to the new encoding.
- **Session pane not reset on new worktree creation** ‚Äî Creating a new worktree left `session_file_path` pointing at the previous worktree's session file. The file watcher would then re-populate `display_events` from the stale file on the next poll cycle. Fixed by clearing `session_file_path`, `session_file_modified`, `session_file_size`, and `session_file_dirty` at the top of `load_session_output()`.
- **Worktree creation dialog rejects valid characters** ‚Äî Input filter was limited to alphanumeric, `-`, `_`, `.` only. Expanded to also allow `+`, `@`, `/`, `!` ‚Äî all valid in git branch names.
- **Worktree creation dialog leaks keybindings** ‚Äî Typing `G` (Shift+G) in the worktree name input opened the Git panel instead of inserting the character. Fixed by routing `WorktreeCreation` and `BranchDialog` focus modes directly to their handlers before keybinding resolution.
- **Auto-rebase skips dirty worktrees** ‚Äî `check_auto_rebase()` now checks `git status --porcelain` before rebasing; worktrees with uncommitted changes are silently skipped instead of failing.
- **Auto-rebase success dialog hidden while viewing another worktree's git panel** ‚Äî `check_auto_rebase()` blanket-skipped all worktrees when the git panel was open. Now only skips the specific worktree whose git panel is being viewed ‚Äî other worktrees still auto-rebase and show the 2-second success toast.
- **Orphaned rebase state cleaned up on startup** ‚Äî On launch, all worktrees are scanned for `.git/rebase-merge/` or `.git/rebase-apply/` directories (left by app crashes during rebase) and auto-aborted.
- **Stash orphaned after merge-triggered RCR** ‚Äî `accept_rcr()` now pops any stash left on main before re-calling `squash_merge_into_main()`, preventing orphaned stash entries.
- **Auto-rebase config not cleaned up on worktree removal** ‚Äî Post-merge dialog archive/delete handlers now remove the `auto-rebase/<branch>` key from `azufig.toml` and the in-memory `HashSet`.
- **Push targets wrong branch / non-fast-forward rejections** ‚Äî `Git::push()` now always resolves `HEAD` explicitly and runs `git pull --rebase origin <branch>` before pushing, preventing both wrong-upstream pushes and non-fast-forward rejections. Rebase conflicts are reported as errors; offline/no-upstream failures are silently skipped (push will create the branch). `Git::pull()` falls back to `git pull origin <branch>` when no upstream is configured. After rebase, `push()` detects diverged branches (`rev-list --left-right --count`) and auto-switches to `--force-with-lease`, skipping the pull-rebase step. Status message shows "(force-pushed)" suffix.
- **Squash merge fails when local main has unpushed merges** ‚Äî `git pull --ff-only` on main during squash merge would bail with "Cannot update main" if local main diverged from remote (e.g., prior squash merge not yet pushed). Made pull always non-fatal ‚Äî the feature branch is already rebased onto main, so the squash merge works regardless.
- **Squash merge fails with "unmerged files"** ‚Äî Orphaned `UU` conflict markers on main (from prior interrupted merges or concurrent worktree work) blocked `git merge --squash`. Pre-flight cleanup now: aborts merge/rebase if state files exist, then resolves any remaining unmerged files with `checkout --ours` + `git add` so the squash merge can proceed.
- **Squash merge fails when main has dirty working tree** ‚Äî `squash_merge_into_main()` now stashes dirty state before merging and pops after commit.
- **Squash merge reports false failure on already-merged branches** ‚Äî `git commit` prints "nothing to commit" to stdout, not stderr. The error check only read stderr, so the `nothing to commit` early-return never triggered ‚Äî instead it bailed with `"Squash merge staged but commit failed: "`. Fixed by checking both stdout and stderr. Also cleans up stale `SQUASH_MSG` files (left by failed merges) during pre-flight and after the nothing-to-commit path.
- **Squash merge with uncommitted changes loses work** ‚Äî `exec_squash_merge()` now blocks if the feature branch has uncommitted changes.
- **RCR session disappears after Claude exits** ‚Äî Fixed by skipping session-file re-parse when the exiting slot's file doesn't exist in the current worktree's session directory.
- **Branch dialog showed no branches** ‚Äî All branches were checked out in worktrees and filtered out. Fixed by showing all branches with checked-out indicators instead of filtering.
- **Branch dialog froze UI** ‚Äî `list_remote_branches` did `git fetch --all --prune` (blocking network call). Replaced with `list_remote_branches_cached` that reads local cache only.
- **Branch dialog created wrong worktree type** ‚Äî Used `Git::create_worktree` (creates new branch with `-b`) instead of `Git::create_worktree_from_branch` for existing branches.
- **Commit log showed all worktrees' commits** ‚Äî `git log` without a range filter followed HEAD's full ancestry, including squash merge commits from other branches that were merged into main. Feature branches now use `git log main..HEAD` to show only branch-specific commits.
- **Rebase replayed squash merge commits from other branches** ‚Äî Plain `git rebase main` could replay other branches' squash merge commits that ended up in the branch's history from a prior rebase. Now uses `git rebase --onto main <fork-point>` to replay only branch-specific commits.
- **Configurable auto-resolve conflicts via union merge** ‚Äî Files that frequently conflict during rebase (docs, config, etc.) can now be auto-resolved using `git merge-file --union` which keeps BOTH sides' changes with zero content loss. Default list: AGENTS.md, CHANGELOG.md, README.md, CLAUDE.md. Press `s` in the Git panel to open an interactive overlay for managing the auto-resolve file list (j/k navigate, Space toggle, `a` add, `d` remove, Esc save). Config persisted per-project in azufig.toml. Replaces the hardcoded `DOC_FILES` constant and `--theirs` strategy.
- **Rebase stashes dirty working tree** ‚Äî `exec_rebase_inner()` now runs `git stash --include-untracked` before rebasing and pops on all exit paths (success, failure, conflict). Prevents "cannot rebase: You have unstaged changes" from files like `.DS_Store` or editor swap files that aren't in the git panel's changed files list.
- **Squash merge stash checks exit code** ‚Äî `squash_merge_into_main()` stash now validates `status.success()` in addition to checking stdout text, preventing silent stash failures from leaving dirty state that blocks `git merge --squash`.
- **Worktrees show as archived during RCR and after detached HEAD** ‚Äî During RCR, the rebase leaves HEAD detached. If `load_worktrees()` ran, it saw `branch: None` ‚Üí blank sidebar entry + real branch appended as archived ghost, and output routing broke (convo pane showed nothing). Fixed with two-case handling: (1) active rebase ‚Üí reads branch from `rebase-merge/head-name` without aborting; (2) orphaned detached HEAD ‚Üí re-attaches via checkout. Also moved repair from startup-only into `load_worktrees()` so every refresh benefits. Belt-and-suspenders: `handle_claude_output()` now always shows output matching the RCR session's slot ID, regardless of `current_worktree().branch_name`.

### Changed
- **Centralized branch prefix** ‚Äî All 9 hardcoded `"azureal/"` branch prefix assumptions replaced with a `BRANCH_PREFIX` constant and `strip_branch_prefix()` helper in `src/models.rs`, making the app fully repo-agnostic. Doc comments updated to use generic `{prefix}/` examples.
- **azufig confirmed project-scoped** ‚Äî Removed aspirational worktree-local azufig documentation (never implemented). The project-local `.azureal/azufig.toml` is always at the main worktree root (resolved via `git rev-parse --git-common-dir` parent), shared by all worktrees. No per-worktree copies.
- **Git status box hints** ‚Äî Removed `a:auto-rebase` from status box footer since it's already listed in the Actions pane.
- **`.DS_Store` untracked from git** ‚Äî File was in `.gitignore` but already tracked in git history, causing it to appear as a conflict offender during RCR. Removed from tracking via `git rm --cached`.
- **Auto-untrack gitignored files** ‚Äî `Git::untrack_gitignored_files()` runs `git ls-files -i --exclude-standard` to find tracked files matching `.gitignore`, then `git rm --cached` to remove them from the index without deleting the working-tree copy. Called on startup (`load()`) and after every `stage_all()` (`git add -A`) to prevent gitignored files from being committed or causing rebase/merge conflicts.

### Removed
- **`‚åÉd` global debug dump shortcut** ‚Äî Debug dump functionality moved into the AZUREAL++ panel (‚åÉa ‚Üí Debug tab ‚Üí Enter). Global `debug_dump_naming` and `debug_dump_saving` state fields removed from App.
